How to Run GULP

From the terminal

OSX: cd /Applications/MAMP/www/
PC: cd (your install location)/MAMP/www/
TYPE: gulp watch

AFTER Local Build

Remove the following files/directory:

  • remove account-name/.hub-welcome-text
  • remove account-name/.metadata

Upload to the HUB via SFTP

  • Upload the 'www' directory and pertinent 'account-name' directory.
  • NOTE: Do not change the current directory structure. ie: images/callouts etc, includes, uploads/images
    NOTE: Do NOT remove the '/assets/includes/data/' directory
  • Finish the build online.
From the terminal:
  • FIRST TIME OR IF THERE HAS BEEN A MAJOR UPDATE
  • (OSX/PC)
  • cd MAMP/www
  • git status
  • git fetch
  • git checkout psds-buildkit- <current version>
  • git pull
  • git branch (should display current version)
  • MINOR UPDATES
  • cd into MAMP/www
  • git pull

KNOWN ERROR MESSAGES

And what to do about them.

When a local account-name directory is deleted, Gulp will throw an error and quit.
Restart gulp and continue.

When you re-visit an existing account-name local buildkit. This error will be visible in the terminal running Gulp.
"... Uhoh. Got error write EPIPE ... Error: write EPIPE at WriteWrap.onWriteComplete [as oncomplete] (internal/stream_base_commons.js:83:16)"
Gulp automatically continues. If not, simply restart Gulp.

Update the SELECT ACCOUNT NAME to start work on the revisited site.
