03-26-2023

Changelog 03/26/2023

CSS/LESS:
  • testimonial slideshow ‘.controls’ element and ‘button’ elements within it have ‘pointer-events’ rules added so that the buttons behave as expected, without the ‘.controls’ element causing an un-targetable band across the slideshow itself on tablet/desktop device widths.
  • testimonial slideshow ‘.controls’ rules updated to reflect ‘.icon’ class on controls vs previously targeting
    element.
  • ALL less ‘@variables’ uniformly placed directly above parent sections of the elements they affect in order to unlock global scope for more powerful/meaningful use opportunities. Previously *some* elements had variables nested within them, most noticeable example is on elements within ‘#contactinfo’ where ‘@addresslh, @addressfs, @phonelh, @phonefs’ used to be enclosed with the ‘.address’ and ‘.phone’ element, ex:
    Used to be like this:
     .address{
                        @addresslh:30;//line-height @addressfs:20;//font-size line-height:@addresslh/@addressfs; font-size:@addressfs/20em; other:rules; et:cetera; } .phone{
                        @phonelh:30;//line-height @phonefs:20;//font-size line-height:@phonelh/@phonefs; font-size:@phonefs/20em; other:rules; et:cetera; }

    Now live above the ‘#contactinfo’ element:
     /* =============================================== contact info =============================================== */ @addresslh:30;//line-height @addressfs:20;//font-size @phonelh:30;//line-height @phonefs:20;//font-size #contactinfo{
                        .address{ other:rules; } .phone{ et:cetera; } }
  • '@bttlineheight' variable added for ‘a#backtotop’ element. It is is still set equal to ‘@linkslineheight’ for common use but in the event that it needs to be different, it can now be adjusted via the variable minimizing the need for re-factoring code within.
  • generic button class ‘intbtn’ added for use by PM/PAs to add quick buttons to pages to streamline the post-build editing process. (Thank you for recommending this Mike)
  • ‘invisalign-videos’ page added to list of centered content pages with .before-after, .office-tour, etc…
  • ’embed-container’ enhancement added for modern browsers improving the rendering and capping the size so they aren’t so visually imposing on larger devices.
LAYOUT:
  • Scripts section updated to make all javaScript es6 standard. this includes:
    • use of arrow functions instead of the ‘function’ keyword where appropriate:
      more about Arrow Functions
    • use of the const keyword to declare constant variables (since they do not need to be re-assigned):
      more about const keyword
    • use of let keyword to declare limited block scoped local variables:
      more about let keyword
  • 'slideshow.php' include corrected - slide4 was duplicated instead of having a slide5.
  • ’fetchpriority’ attribute added to all img tags as they no longer flag errors in w3c validation (after almost exactly an entire year!)
    more about priority hints
  • 'GA code' publication has now been automated, i.e. the builder no longer needs to uncomment the section; If the '$gacode' field is populated on the site manager or posthumously at the top of layout.php, the GA code script will automatically activate.
  • 'mediaslider' control's icons have ‘icon’ class added and FA font-weight specific class ‘fa-solid’ added
JS:
  • lazysizes updated to v5.3.2.
  • curated version of 'modernizer.js' added to '/concat-js-files/' directory to be included as standard until Sesame officially stops supporting ie11. This is necessary to ease support for es6 standards.
    Instructions on how to use this to target ie11 more easily via Javascript and css:
    • CSS - modernizr adds a class of ’.appearance’ to the 
      html
      elements if its not ie11 and ‘,no-apeparance’ if it is ie11:
       .no-appearance .box  {to target ie11 only. } .appearance .box { to hide your cutting edge styling from expired non-compliant browsers that shouldn’t even be out there for use anymore. }
    • JS - javascript can be used to target non-ie11 browsers utilizing the same ‘appearance’ keyword:
       if (Modernizr.appearance) { // supported } else { // not-supported }
  • local.js:
    parallax script for slideshow upgraded:
    1. to include `will-change: transform` when transforming to improve performance, and
    2. only activates on desktop devices so that it doesn’t highjack styling that occurs mobile devices.
