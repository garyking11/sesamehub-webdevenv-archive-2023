Tools and Utilities

Create Buildkit

This is the process to create the initial buildkit.

  1. In your terminal, type cd into your install location:
  2. - macOS: /Applications/MAMP/www
  3. - PC: c:/MAMP/www
  4. In your terminal, type gulp watch, then enter
  5. At the top of the page ADD THE ACCOUNT-NAME of the website you are building in the Create Directory input field ie: jones-bob.
  6. CLICK CREATE DIRECTORY
  7. Using Select Account dropdown located under Select Account, select the account name that you have just entered.
  8. CLICK SELECT ACCOUNT ie: account-name
  9. ADD builder name
  10. After creating the directory and selecting the account, run 'gulp watch' in your terminal
  11. If there is an error in the terminal, restart gulp by typing 'gulp watch'
  • NOTE: If you are unable to retrieve main-nav or welcome text, upload the new layout.php to the account-name you are working on.

    • Hub Welcome Text

    This button will load the welcome text as defined within the Sesame Hub Dashboard into your buildkit. It is also editable within the textarea containing the markup.

    • This account-name must be present on the HUB!
    • Alternatively if you do not have an internet connection you can use "STOCK welcome text."
    • Click Apply after content is loaded.

    HUB #main-nav

    This button will load the current hub sitemap as defined within the Sesame Hub Dashboard into your main-nav html within the buildkit. Alternatively if you do not have a internet connection you can use "STOCK #main-nav."

    • Click Apply after content is loaded

    Select Options

    • Add the GA CODE in the provided form field
    • Add the tagline for slideshow or video (if present)
    • This is the tagline this is positioned over the Hero (HTML allowed)

    Add the following components to buildkit

    • Hero - Select either the "classic" or "video"
    • Content (required). Select Top Subnav or Side Subnav
    • Add Spotlights After reviewing the design select the appropriate layout
      • Spotlight One Horizontal
      • Spotlight Two Horizontal
      • Spotlight Three Horizontal
      • Spotlight FourHorizontal
      • Spotlight Five Horizontal
      • Spotlight Column Img Left
      • Spotlight Column img Right
      • Testimonial Rotator
      • Association Logos
    • Only the Spotlights and Testimonial Rotators are allowed to have more than one item. If for example you want to have 2 slideshows in the build, add it manually.

    Restart Gulp! (important) ERROR TIP: if Gulp quits with an error, type control C from the active terminal and restart Gulp by typing gulp-watch in the terminal.

    webP converter

    The site manager has a built-in webP converter. This function will automatically convert images placed into their appropriate directories ie: assets/images/office-tour, assets/before-after-cycle, assets/ uploads/images/ etc.
    A new directory will be created in the assets/ directory with the newly created webP images. These can then be uploaded to the hub.

    Save/Finish

    WARNING: If you click the SAVE/FINISH' button after the site has been created and you perhaps have done some work, the existing buildkit with that account-name will be overwritten with a new buildkit.

    Ready to build ...

    • Navigate to the newly created starter buildkit
    • Go to the newly created browser tab and hard refresh the starter buildkit
      Mac: http"//localhost:8888/account-name/#/
      PC: http"//localhost/account-name/#/
    • While Gulp is running your buildkit changes ie: .less and html markup will automatically be updated and be visible on your machine in the starter buildkit.
    • The page will then reload automatically to show the changes you have made.

    After the build is completed locally Upload the 'www' directory to the account-name/ on the hub.

    DELETE AND REPLACE THE HUB 'www' DIRECTORY with the files in your buildkit.

    DO NOT UPLOAD the following files/directory:

    • .metadata
    • .hub-welcome-text
    • Upload the 'www' directory and it's contents to the account-name on the hub.
    • Finish the build on the HUB.
    • NOTE: Do not change the current directory structure. ie: images/callouts etc, includes, uploads/images.
    • NOTE: Do not remove the /assets/includes/data directory or it's contents.
    • NOTE: You can reset welcome text and main-nav text only after you have clicked 'Apply' once during the creation of the template.
    • Sub navigation menus are hard coded and are not pulled from the hub.

    RE-STARTING A PROJECT

    You can restart the build by clicking the 'SAVE/FINISH' button. This will overwrite your current account-name directory and any work you have done.

    KNOWN ERRORS:

    "... Uhoh. Got error write EPIPE ... Error: write EPIPE at WriteWrap.onWriteComplete [as oncomplete] (internal/stream_base_commons.js:83:16)" gulp automatically continues and you can continue also.

    Restart gulp to fix these errors. (control C, gulp watch) When a local account-name directory is deleted, gulp will throw an error and quit. Restart gulp to fix this error. type control C, gulp watch from the terminal.

    When you re-visit an existing account-name local buildkit. This error will be visible in the terminal running gulp. type control C, gulp watch from the terminal

    .less syntax errors may cause Gulp to quit and fail. Usually though, the error will show up in the terminal and direct you to the syntax error. However, if Gulp fails and stops, Fix the error and restart Gulp. type control C, gulp watch from the terminal

     Image Manager

    The image manager will allow you to drag and drop images into the appropriate directories.

    • Use the dropdown menu to select and copy where you want the image placed.
    • Drag and drop or click to select and upload from the directory where your image are located.
    • webP images will automatically be created and placed in the assets directory
     Stock Photo Manager

    This page will add images to assets/uploads/images. These images are used for internal pages.

    1. Select desired ratio (1x1, 3x4 etc)
    2. Select desired category (adult, braces etc)
    3. Select library page
    4. Select image to associate with library page
    5. Repeat ...
    6. Your selected image will be copied into the local buildkit. (account-name/assets/uploads/images/)

    Builder Notes

    The Site Manager now has a place to store snippets of code that you may want to use in the future. You will be able to add, save and delete notes and snippets.

    • Create (Add) a snippet item in the Builder Notes page and save your snippet. This will be stored for future use.
    • There is a backup made for these snippets to prevent mistakenly overwriting this file. It is located under MAMP/data/builder-data.json
    Form Builder

    The form builder can be used to create custom forms requested by the producer.

    • Click the Add Form Field Button
    • Add the page name as it will be named in the HUB dashboard.
    • Select the desired form type. ie: text, select, textarea, checkbox.
    • Add the field name.
    • Special characters are not allowed and the field name will not allow them to be entered.
    • Add the Field label -- Special characters ARE allowed.
    • Select required if desired by checking the 'Required' checkbox.
    • Form fields can be deleted using the "Delete Form Field." text.
    • Select form fields allow the builder to add and delete selectable values.
    • All formatting is done automatically. There is no need to alter the provided HTML.
    • Form fields are sortable. Sort the order of the form fields desired by dragging and dropping the fields.
    • Click Grab your HTML -- This will copy the HTML markup to the clipboard.
    • Place the HTML into the appropriate page in the HUB dashboard.
